# Link Manager

A web application for saving, organising and searching useful links.

The application contains the following features:

* Users can create an account, log in and log out
* Users can add, edit and delete their own links
* Users can view links added by all users to the application
* Users can search for links by title, URL, notes or category
* Users can assign one optional primary category to a link
* Users can add comments containing additional information to links
* Users have profile pages and can view other users' profile pages with user activity statistics and links added by the user

## Guide for testing

Run the following commands. Make sure Python 3 and sqlite3 are installed first:

```
git clone https://github.com/saanaol/link-manager
cd link-manager
python3 -m venv venv
source venv/bin/activate
pip install flask
sqlite3 database.db < schema.sql
sqlite3 database.db < init.sql
flask run
```

Then open the application in your browser.

In the application, you can test the following:

* Register a new user
* Log in and log out
* Add a new link and select a primary category for it
* Check that the added link and related information are shown on the link page
* Search for a link by title, URL, notes or category and return to the full link list
* Open a user profile page by clicking the username shown below a link
* Check the user activity stats and links added by that user through the user profile page
* Add a comment to a link
* Check that added comments are shown below the related link
* Edit and delete your own links

## Large data testing

The application was tested with a large SQLite database generated with `seed.py`.

For large data handling, pagination is used for:

* link listings
* link comments
* search results
* links added by a user

The database also includes indices to improve performance with a large amount of data.

The large dataset generated by `seed.py` contains:

* 1,000 users
* 1,000,000 links
* 10,000,000 comments
* user and link edge cases with a substantial amount of user activity

The database was created with the following commands:

```
rm database.db
sqlite3 database.db < schema.sql
sqlite3 database.db < init.sql
python seed.py
```

After creating the large test database, I tested the following pages:

| Page                       | Loading time |
| -------------------------- | ------------ |
| `/links/1`                 | 0.01 s       |
| `/links/2`                 | 0.01 s       |
| `/links/100`               | 0.01 s       |
| `/link/1`                  | 0.01 s       |
| `/link/1?page=1`           | < 0.01 s     |
| `/link/1?page=2`           | < 0.01 s     |
| `/search_links?query=test` | 0.14 s       |
| `/user/1`                  | < 0.01 s     |

Results: The application and the tested pages remained usable with a large test database, and the link listing and search results loaded normally with a large number of links. A link with 10,000 comments and a user profile with 5,000 added links were tested as edge cases, and both remained usable.
